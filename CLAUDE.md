# CardOps marketing site

Static, no-build marketing site for **CardOps** — a pre-launch iOS TCG inventory app for trading-card vendors who sell at shows. Plain HTML/CSS: `index.html`, `privacy.html`, `terms.html`, `style.css`.

## Design Context

Full strategy: see [PRODUCT.md](PRODUCT.md). Quick reference for any design work here:

- **Register:** brand (marketing/landing — design IS the product). **Platform:** web.
- **Primary audience:** full-time / serious card dealers who run tables at shows.
- **Positioning:** the inventory app built for vendors who *sell at shows* — offline-first, per-show profit, made for the table not the desk.
- **Personality:** no-nonsense pro tool — dependable, fast, straight-talking. Trust and relief, never hype.
- **Anti-references (stay out of all four):** generic SaaS template (gradient hero + feature-card grid), crypto/hype, childish TCG clip-art, corporate/sterile B2B.
- **Known gap:** the current `style.css` (indigo→ink gradient hero + white feature-card grid) sits squarely in the #1 anti-reference lane. An overhaul is planned; capture `DESIGN.md` via `/impeccable document` *after* the new visual system lands, not before.
- **A11y target:** WCAG AA — mind muted text contrast on the cream body.
